# About Shilp Sutra — शिल्प सूत्र

*A Design System by Devalok Design & Strategy Studios*

---

## Philosophy

Shilp Sutra is built on the belief that design systems should be opinionated about composition, not just styling. Every component embodies three principles:

1. **Accessible by default** — WCAG AA compliance baked into every component. Consumers get correctness without configuration.
2. **Composable by design** — Compound components, slot patterns, and `asChild` ensure components compose predictably.
3. **Culturally grounded** — Rooted in the warmth and precision of Indian craft traditions, reflected in our color language and design sensibility.

---

## Architecture

### Three-Tier Token Architecture

```
Tier 1: Primitives (primitives.css)
  Raw palette values — never used directly in components.
        ↓
Tier 2: Semantic (semantic.css)
  Intent-based tokens — what components consume.
        ↓
Tier 3: Tailwind Preset (preset.ts)
  Utility classes mapped from semantic tokens.
        ↓
Components (CVA variants + cn())
```

This layered approach separates raw values from intent-based decisions, making theming and dark mode straightforward. Toggle the `.dark` class on the root element and every token responds automatically.

### Module Hierarchy

```
ui/ (primitives)  →  shared/ (composites)  →  layout/ (shells)  →  karm/ (domain)
```

Each layer can only import from layers to its left. This ensures the design system core remains independent of domain logic.

## Packages

Shilp Sutra is distributed as three npm packages under the `@devalok` scope:

| Package | Description |
|---------|-------------|
| `@devalok/shilp-sutra` | Core component library — UI primitives, shared composites, layout shells, and design tokens |
| `@devalok/shilp-sutra-karm` | Domain-specific components for project management — boards, tasks, chat, dashboards, and admin tools |
| `@devalok/shilp-sutra-tailwind` | Tailwind CSS preset that maps semantic tokens to utility classes for consistent theming |
