# UI Primitives

Shilp Sutra includes **64+ low-level UI primitives** built on top of vendored [Radix UI](https://radix-ui.com). These are the foundational building blocks used by all higher-level components in the system.

Every primitive:

- Uses **semantic CSS tokens** for consistent theming
- Supports **dark mode** out of the box
- Uses **forwardRef** for composability
- Follows WAI-ARIA patterns via Radix UI
- Uses **CVA** for type-safe variant/color props and **cn()** for className merging

```tsx
import { Button, Input, Dialog } from '@devalok/shilp-sutra/ui'
```

## Key Patterns

### Two-Axis Variant System

Most components accept **`variant`** (visual style) and **`color`** (semantic meaning) as separate props:

```tsx
<Button variant="solid" color="error">Delete</Button>
<Badge variant="subtle" color="success">Active</Badge>
```

### Server Components

Some components are server-safe (no `"use client"` directive). Import individually:

```tsx
import { Text } from '@devalok/shilp-sutra/ui/text'
import { Skeleton } from '@devalok/shilp-sutra/ui/skeleton'
```
