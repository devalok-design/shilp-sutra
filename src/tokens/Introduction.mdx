# Design Tokens

Shilp Sutra uses a **three-tier token architecture** that separates raw values from intent-based decisions, making theming and dark mode straightforward.

```
Tier 1: Primitives    Raw palette values (never used directly in components)
Tier 2: Semantic       Intent-based tokens (what components consume)
Tier 3: Tailwind       Utility classes mapped from semantic tokens
```

---

## Tier 1 -- Primitives

Raw color palettes defined as CSS custom properties. Each palette provides a 50-950 scale.

| Palette | Role | Scale |
|---------|------|-------|
| **Pink** | Brand / interactive | 50 -- 950 |
| **Purple** | Secondary accent | 50 -- 950 |
| **Neutral** | Grays (warm-tinted) | 0, 50 -- 950 |
| **Green** | Success | 50 -- 950 |
| **Red** | Error / danger | 50 -- 950 |
| **Yellow** | Warning | 50 -- 950 |
| **Blue** | Info / links | 50 -- 950 |

Example: `--pink-500: #d33163`, `--neutral-900: #282425`

## Dark mode

All semantic tokens have `.dark` class overrides. Toggle dark mode by adding the `dark` class to any parent element:

```css
.dark {
  --color-background: var(--neutral-950);
  --color-text-primary: var(--neutral-50);
  /* ... all tokens remap automatically */
}
```

No component code changes are needed -- the token layer handles the entire theme switch.
